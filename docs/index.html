OopsQL

Find out what your query can mess up before you run it.

A lightweight T-SQL blast-radius analyzer for developers, analysts, DBAs, ERP admins, and data teams.

What it catches

Risky scripts before they hit production.

Critical

Missing filters

Flags `UPDATE` and `DELETE` statements without a `WHERE` clause.

High

Protected tables

Calls out changes to tables like invoices, payments, clients, and inventory.

Medium

Rollback gaps

Warns when risky scripts do not include rollback notes or a rollback section.

Medium

Reporting traps

Flags `SELECT *`, `NOLOCK`, and possible one-to-many join mistakes.

Install

Run it from the command line.

git clone https://github.com/75link/OopsQL.git
cd OopsQL
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

oopsql scan examples/dangerous_update.sql

No database connection needed.

OopsQL reads `.sql` files and reports risk. It does not connect to SQL Server and does not execute scripts.

Config

Mark the tables that matter most.

Add an `oopsql.yml` file to set protected tables and transaction rules for your project. 

protected_tables:
  - invoice
  - payment
  - inventory
  - client

require_transaction_for:
  - UPDATE
  - DELETE
  - MERGE
  - INSERT
  - ALTER
  - DROP
  - TRUNCATE
OopsQL Static analysis only. No SQL execution.
